RQ 1) The user will be able to use the system by:
RQ 1.1) Authenticating with an email and a password.
RQ 1.2) Accessing the application without logging in, continuing as a "guest".
RQ 2) The user will be able to register in the system by providing:
RQ 2.1) A username.
RQ 2.2) An email address that is not currently in use.
RQ 2.3) A password or access key for its account.
RQ 2.4) A password confirmation, which must match the provided password.
RQ 3) In the application, the user will be able to:
RQ 3.1) Upload a JPG photo of the sky using the mobile app to be analyzed.
RQ 3.1.1) The user will be able to take a photo with the device's camera.
RQ 3.1.2) The user will be able to select a photo from the gallery or the device's file system.
RQ 3.1.3) The user will be able to decide whether or not to enable explainability for the analysis.
RQ 3.2) If the analysis has not yet been completed, the user will be able to cancel it to prevent the image processing.
RQ 3.3) If the analysis has already been completed, the user will be able to see the results of the photo analysis, which will include:
RQ 3.3.1) The submitted image along with identified clouds:
RQ 3.3.1.1) The clouds will be identified by differentiating between 10 types divided into four categories based on their altitude:
RQ 3.3.1.1.1) High-level clouds (above 6000 meters): Cirrus (Ci), Cirrostratus (Cs), and Cirrocumulus (Cc).
RQ 3.3.1.1.2) Mid-level clouds (2000 to 6000 meters): Altostratus (As) and Altocumulus (Ac).
RQ 3.3.1.1.3) Low-level clouds (up to 2000 meters): Stratus (St), Stratocumulus (Sc), and Nimbostratus (Ns).
RQ 3.3.1.1.4) Clouds with vertical development (spanning multiple layers at once): Cumulus (Cu) and Cumulonimbus (Cb).
RQ 3.3.1.1.5) Airplane contrails visible in the image will also be identified.
RQ 3.3.1.2) Clouds will be identified across the entire image, without differentiating between individual clouds and cloud formations.
RQ 3.3.2) A weather forecast for the next 2-3 hours based on the clouds identified in the image:
RQ 3.3.2.1) Stable weather when no clouds are present or there are mostly Cirrus or Stratocumulus clouds.
RQ 3.3.2.2) Possibility of vertical development when Cumulus clouds are present.
RQ 3.3.2.3) Low visibility when Stratus clouds are present.
RQ 3.3.2.4) Precipitation when Nimbostratus clouds are detected.
RQ 3.3.2.5) Storms when Cumulonimbus clouds are detected.
RQ 3.3.2.6) Weather change when Altostratus clouds are present.
RQ 3.3.2.7) High-altitude weather instability when Altocumulus, Cirrocumulus, or airplane contrails are detected.
RQ 3.3.2.8) Warm front when Cirrostratus clouds are present.
RQ 3.3.3) The user will also receive a forecast warning if the identified clouds have any potential risk, which include: storms, thunderstorms, showers, persistent fog, persistent precipitation, and imminent weather changes.
RQ 3.4) The user will be able to view the history of previously analyzed images:
RQ 3.4.1) They will be able to see a list of their previous analyses, with their status, the date and time the analysis was performed, and, if available, the location from which it was performed. Each analysis can have three statuses:
RQ 3.4.1.1) In progress.
RQ 3.4.1.2) Canceled.
RQ 3.4.1.3) Completed.
RQ 3.4.2) They will be able to access each analysis from the list and view its results, including: the analyzed image, the status of the analysis, the date and time of the analysis, the location from which it was performed (if available), the identified clouds, the forecast, and the associated warnings.
RQ 3.4.2.1) If explainability was enabled for the analysis, the user will be able to see the bounding boxes of each cloud superimposed on the analyzed image, with a label indicating the type of cloud detected.
